React開発へようこそ！

このガイドは、Pythonでの開発経験を持つあなたが、Reactの標準的なプロジェクト構造をスムーズに理解できるよう設計されています。左側のファイルツリーをクリックすると、各項目にジャンプできます。なぜこのような構造が採用されているのか、その背景にある「思想」まで掘り下げていきましょう。

プロジェクトの全体像

まずは典型的なReactプロジェクトのフォルダ構成です。この構造は、プロジェクトが成長しても破綻しにくい、拡張性の高い構成になっています。

node_modules/

プロジェクトが依存するすべてのライブラリ（パッケージ）がインストールされるフォルダです。`npm install`コマンドで`package.json`を元に自動的に生成・更新されます。

🐍 Pythonとの対比

Pythonの仮想環境（`venv`）内の`site-packages`フォルダに相当します。このフォルダは直接編集せず、Gitの管理対象からも外します。

public/

ここに置かれたファイルは、Reactのビルドプロセスを経ずに、そのままの形でWebサーバーに公開されます。静的なアセットを置く場所です。

特に重要なのが `index.html` です。これはReactアプリケーションの「土台」となる唯一のHTMLファイルで、この中の`

`のような要素に対して、Reactが動的にUIを描画していきます。これを**シングルページアプリケーション(SPA)**と呼びます。

src/

開発者が書くコードは、すべてこの`src`フォルダの中に置かれます。Pythonプロジェクトで言えば、アプリケーションの本体となるパッケージディレクトリに相当します。この中の構成が、プロジェクトの保守性を大きく左右します。

設計思想: 関心の分離

`src`フォルダを細かく分けるのは「関心の分離」という思想に基づきます。役割の異なるコードを別々の場所に置くことで、コードの見通しを良くし、変更や再利用を容易にします。

src/api/

外部のAPIサーバーとの通信に関する処理をまとめる場所です。データの取得（GET）、作成（POST）などを行う関数をここに置きます。コンポーネントが直接API通信の詳細を知らなくても済むようにするのが目的です。

src/assets/

画像、フォント、グローバルなCSSファイルなど、ビルドプロセスに含めたいアセットファイルを置きます。JavaScriptファイル内で `import` して使用します。

src/components/

再利用可能なUI部品を置く場所です。ボタン、入力フォーム、カードなど、アプリケーションの様々な場所で使い回す小さなUIパーツをここにまとめます。`common/`（原子的な部品）や`layout/`（骨格部品）のように、さらに細分化することもあります。

src/contexts/

アプリケーションの広範囲で共有したい「グローバルな状態」を管理する場所です。例えば、ログインしているユーザー情報やサイト全体のテーマ（ライト/ダークモード）などです。これにより、コンポーネントの階層が深くても、propsを介さずにデータにアクセスできます（プロップスのバケツリレー問題の解決）。

src/hooks/

ロジックを再利用するための置き場所です。Reactでは「フック」という仕組みを使って、UIの見た目から状態管理やAPI通信などの「ロジック」を切り離すことができます。複数のコンポーネントで同じロジックが必要になった場合、それをカスタムフック（`use`で始まる関数）としてこのフォルダに切り出します。

src/pages/

アプリケーションの「ページ」に相当する、大きな単位のコンポーネントを置きます。「ホームページ」「ユーザープロフィールページ」などがこれにあたります。`pages`のコンポーネントは、`components`フォルダにある小さな部品を組み合わせて作られます。

src/utils/

特定の機能に依存しない、汎用的な便利関数を置きます。日付のフォーマット、金額のカンマ区切りなど、プロジェクトのどこからでも使えるヘルパー関数です。

🐍 Pythonとの対比

Pythonプロジェクトにおける`utils.py`や`helpers.py`と全く同じ役割です。

src/App.jsx

**ルートコンポーネント**。アプリケーション全体の「親」となるコンポーネントです。ヘッダーやフッターのような全ページ共通のレイアウトや、ページの切り替え（ルーティング）の定義などをここで行うことが多いです。

src/index.jsx

**エントリーポイント**、つまりアプリケーションの「起動ファイル」です。`public/index.html`の`#root`要素と、メインコンポーネントである`App.jsx`を結びつける役割を持ちます。

// index.jsx の中身の例
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

const rootElement = document.getElementById('root');
const root = ReactDOM.createRoot(rootElement);

root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);

.gitignore

Gitのバージョン管理から除外するファイルやフォルダを指定します。`node_modules`やビルド生成物、環境変数ファイル（`.env`）などが一般的に含まれます。

package.json

プロジェクトの「戸籍」のようなファイル。名前、バージョンといったメタデータに加え、依存ライブラリの一覧や、`npm start`のような開発用コマンドを定義します。

🐍 Pythonとの対比

`requirements.txt`や`pyproject.toml`に非常に近いです。

package-lock.json

プロジェクトの依存関係の「完全なスナップショット」です。実際にインストールされたライブラリの正確なバージョンが記録され、「自分のPCでは動いたのに…」という問題を未然に防ぐ、チーム開発における生命線です。

まとめ：思想と関連性

このフォルダ構成の根底にあるのは、**「関心の分離」**という一貫した思想です。見た目、ロジック、外部との通信などを分離することで、コードの可読性、保守性、再利用性が向上します。この基本形を理解しておけば、どんなReactプロジェクトを見ても、その構造と意図を素早く把握できるようになるはずです。

デジタル庁デザインシステム実践ガイドライン

このガイドラインは、デジタル庁が公開しているデザインシステム（β版 v2.6.0）をベースに、実際のプロジェクト開発で活用できるよう体系化したものです。行政機関や公共性の高い組織のウェブサイト・アプリケーション開発における品質向上と効率化を支援します。

重要方針: 「誰一人取り残されない、人に優しいデジタル化」の実現のため、アクセシビリティを最優先事項として位置付けています。

主要な特徴

  • 実用性重視: 実際の開発で使用できる具体的な仕様とコードサンプル
  • アクセシビリティ準拠: WCAG 2.1/2.2、JIS X 8341-3:2016 準拠
  • レスポンシブ対応: モバイルファーストの設計思想
  • 継続的な改善: 公式デザインシステムの更新に対応
  • 境界オブジェクト: 行政部門、外部企業、開発者が共通言語でデザインを議論
  • 汎用性: 行政組織以外でも利用可能な中立的なデザイン

デザインシステムバージョン

現在のバージョン: β版 v2.6.0

カラーシステム

デジタル庁デザインシステムでは、以下の3層構造でカラーを管理し、アクセシブルで一貫性のあるカラーパレットを提供しています。

カラーの基本原則

デザインシステムのカラーは、同じ色相から明度差や彩度差で派生させることで、UI全体の統一感を維持しています。すべての色は背景とのコントラスト比を厳密に管理し、アクセシビリティ要件を満たしています。

テキストスタイル規約

スタイル名は Std-17N-170 のように命名されます：

  • 最初の数字: フォントサイズ (17px)
  • 中間の文字: ウェイト (N=Normal, B=Bold)
  • 最後の数字: 行間 (170%)

行間設定

行間バリエーション

line-height: 1.2 (Tight)

タイトルや見出しなど、コンパクトな表示が必要な場面で使用。視覚的なインパクトを重視する際に適している。

line-height: 1.5 (Normal) - 推奨

本文テキストに最適な行間設定。読みやすさと文字密度のバランスが取れており、長文の読書に適している。

line-height: 1.75 (Relaxed)

ゆったりとした読み心地を提供。アクセシビリティを重視する場面や、高齢者向けのインターフェースに適している。

レイアウトシステム

グリッドシステム

12カラムグリッド

ページの左右マージン、カラム幅、カラム間のガターを統一した12カラムグリッドを基本とします。

  • デスクトップ: 左右に余白を設けて12等分
  • モバイル: 1列構成に変化（レスポンシブ対応）

ブレークポイント

  • 768px以上: 複数カラムレイアウト
  • 768px未満: 1カラムレイアウト

スペーシング

基準単位: 8px - すべての余白やガターは8pxの倍数で設定

  • 8px, 16px, 24px, 32px... の倍数で余白を設定
  • 小さな余白: 密接に関連する要素間
  • 大きな余白: セクション区切り

クリック領域

ボタンやリンクのタップターゲットは 44px以上 を確保（指でも押しやすく）

アクセシビリティガイドライン

基本方針

「誰一人取り残されない、人に優しいデジタル化」の実現のため、以下の要件を必須とします：

コントラスト比

  • テキスト: 4.5:1 以上
  • UI要素: 3:1 以上
  • 大きなテキスト: 3:1 以上

キーボード操作

  • 全ての機能をキーボードで操作可能
  • フォーカスインジケーターの明確な表示（黒い太線＋黄色のリング）
  • 論理的なタブオーダー
  • フォーカス順序とDOM構造の一致

スクリーンリーダー対応

  • 適切なセマンティックHTML
  • ARIA属性の適切な使用
  • alt属性の設定
  • ページ構造の明確化

チェックリスト

☑ カラーコントラスト比の確認（WCAG 2.1/2.2準拠）
☑ フォントサイズの可読性検証（最小16px）
☑ キーボードナビゲーション動作確認
☑ スクリーンリーダーでの読み上げテスト
☑ フォーカス状態の視覚的表示確認
☑ タップターゲット44px以上の確保
☑ 色だけに頼らない情報伝達

実装ガイド

CSS Custom Properties

デザインシステムで使用するCSS変数の例：

:root {
  --primary-blue: #0066cc;
  --font-family-primary: 'Noto Sans JP', sans-serif;
  --font-size-16: 1rem;
  --spacing-md: 1rem;
  --border-color: #dee2e6;
}

開発フロー

  1. デザインシステムの理解: 本ガイドラインの熟読
  2. コンポーネント選定: 目的に応じた適切なコンポーネント選択
  3. カスタマイズ: プロジェクト要件に応じた調整
  4. アクセシビリティ検証: 実装後の品質確認
  5. ユーザビリティテスト: 実際の使用感の検証

Tailwind CSS統合

公式プラグイン

デジタル庁は @digital-go-jp/tailwind-theme-plugin を提供しており、Tailwind CSS v3/v4で利用可能です。

npm install @digital-go-jp/tailwind-theme-plugin

// tailwind.config.js
module.exports = {
  plugins: [require('@digital-go-jp/tailwind-theme-plugin')],
}
